SquareSpace vs. Wordpress

SquareSpace is focused on users without web dev skills. It also has built in e-commerce options and lots of other functionality which would require plugins in Wordpress.


Wordpress is infinitely customizable, but also often used by those without web dev skills. It's technically free but to get similar functionality to SquareSpace users may need to pay for themes, plugins, etc.

Ionic Framework

The Ionic Framework is a web/mobile framework designed to build cross platform applications.

Ionic is built with Angular. Most angular principles can get passed through.

Ionic CLI

Similar to the Angular CLI. You can use it to generate your project, your files, publishing, and everything else.

UI Components

Ionic supports all the standard mobile components we've seen in the past. Ionic uses it's own component tags for these. 



    Username

ion-item tag supports mobile events such as swipe and tap.

Ionic Pro

Ionic Pro offers additional features such as:

  • Send testers an email with a link, they can leave feedback in dashboard
  • Push out code fixes to users without going through app store
  • Activity Log

Theming

Theming is similar to Bootstrap's theming. You can modify source variables to create your own feel.

Ionic has a built in IOS, Android, Windows, and Core theming system. It can be as simple as adding an import statement in your code.

Watch this into on ionic and theming

Icons Library - Ionicons

Ionicons includes 3 versions of most icons. iOS, iOS-Outline, and Material Design

Ionic Native

Ionic leverages plugins to interact with Native Phone components. Apple Paid, Google Maps, BlueTooth, Etc.

Deploying an App

Ionic leverages Cordova to produce builds for phones. You'll need to download the Android SDK and iOS kit to put your app on a mobile device.

Note: You must use a mac to compile an iOS app.

Unit Testing

Types of Software Testing

  • Unit Testing
  • Integration Testing
  • Performance Testing
  • Acceptance Testing
  • Regression Testing
  • E2E Testing

Test Driven Development

TDD is the development process of working in small cycles. Each cycle, you write a test, write the amount of code needed, and repeat.

Unit Testing

Take a small block of code and make sure it works as expected. Unit testing is one of the types of testing typically done in Test Driven Development.

Unit Testing in JavaScript

You need two things:

A runner: Karma

starts a web server, serves up the page that will run the tests, report results of the test

A Testing Framework: Jasmine

Gives you the syntax for writing tests

Basic Syntax


describe("A suite is just a function", function() {
    var a;
    
    it("and so is a spec", function() {
        a = true;
    
        expect(a).toBe(true);
    });
});

Write the Failing Test


it('it should start at 0 and end at 3', inject([PagingService], (service: PagingService) => {
    const pager = service.getPager(10, 1, 4);
    expect(pager.startingIndex).toBe(0);
    expect(pager.endingIndex).toBe(3);
}));

Another Test 


it('it should have 4 pages', inject([PagingService], (service: PagingService) => {
    const pager = service.getPager(10, 1, 3);
    expect(pager.totalPages).toBe(3);
}));
    

it('it should not have go past the total number', inject([PagingService], (service: PagingService) => {
    const pager = service.getPager(10, 10, 3);
    expect(pager.totalPages).toBe(3);
}));

Overall Benefits of Unit Testing

  • Find bugs early
  • Provides basic documentation
  • Helps debugging
  • Encourages good code design
  • Reduces long term cost

Publish Angular App to Github Pages

https://github.com/angular/angular-cli/wiki/stories-github-pages

Steps

  • Step 1 - create repo in github
  • Step 2 - Create your app using Angular CLI just like the last few weeks, make sure you do it in your local git repo
  • Step 3 - After creating your app, run the following in your local project folder ng build --prod --output-path docs --base-href
  • Step 4 - Push your changes and enable GitHub pages
  • Step 5 - Run your app, for example mine is at https://jon20111.github.io/uagc-classwork/src/taco-truck/docs/index.html
